06 Teach : Team Activity

Special Instructions for Online Students

You need to coordinate a one hour synchronous meeting with your group for this activity.

Overview

Last week, we created a database to hold scriptures and a simple website to display them. This week, we will create a database that can hold scriptures and topics, and link scriptures to topics.

Instructions

Add to your scripture DB from last week. If you no longer have that database, please create a table for scriptures with columns for id, book, chapter, verse, and content. You are welcome to refer to the instructor's solution for last week's team activity for help in creating this table.

Core Requirements

  1. Create a table called topic with two columns: id and name. Then, manually insert into this table the following topics: Faith, Sacrifice, Charity.

    Create another table to link scriptures to topics. (This is a many-to-many relationship, so it requires another table for the linking). Make sure to include foreign keys constraints.

  2. Create a PHP page to insert a new scripture along with its topics. (Note that you are inserting a new scripture and adding a link to an existing topic, all in the same POST)

    Show a text box for:

    • Book

    • Chapter

    • Verse

    • Content (The text of the scripture. A textarea would probably be the best choice here).

    Then, have the PHP script generate checkboxes for each topic contained in the DB table.

    Finally, have a submit button to insert the scripture into the database, and connect it to the selected topics.

  3. After a user submits the form, have the application show a page that lists all the scriptures in the database, each one with it's associated topics.

    Using your PHP page, insert the following scriptures into the database along with the applicable topics:

    • Hebrews 11:4

    • 1 Corinthians 13:13

    • Moroni 7:47

Stretch Challenges

After finishing the core requirements, ensure that everyone is at that point and understands the material. When everyone has completed the core requirements, you can move on to these stretch challenges.

Concerning this week's stretch challenges...

Please don't move on to these until you have complete the core requirements.

For those that have experience, it may seem natural to implement these from the beginning (especially combining the submission and results pages). However, for students who are doing this for the first time, it can be a little confusing. This is the reason the activity is design the way that it is.

Similarly, please work through these in order, completing #2 before switching to #3. They will cause you to play with different approaches.

  1. Create another checkbox at the end of the list, with an empty text box next to it. If the user selects this checkbox, add a new topic with the corresponding name to the database in addition to adding the scripture, and the link between them.

  2. Have the list of previously entered scriptures, along with their topics, display on the same page as the "new scripture entry" portion described above. When a new scripture is entered, refresh the page to show the newly entered scripture along with the previous ones.

  3. Complete the previous stretch challenge without causing a page refresh (i.e., use AJAX to complete the submission and update).
