Heroku CLI

The Heroku Command Line Interface (CLI) is a tool for creating and managing Heroku applications.

Install the Heroku CLI by following the installation instructions on the Heroku CLI page. (Just install the CLI for now, we will be creating the application in future steps.)

Verify your installation by running the version command and login with the credentials you used to create your Heroku account.


$ heroku -v
heroku-cli/6.15.25-635e5c6 (darwin-x64) node-v9.5.0
    
$ heroku login
Enter your Heroku credentials.
Email: lyonr@byui.edu
Password (typing will be hidden):
Authentication successful.

PHP 5.6

For some of the Heroku processes, you need PHP installed on your local machine. This was included with your WAPP/MAPP/LAPP install earlier. Please verify that it is installed correctly by running the version command


$ php -v
PHP 5.6.25 (cli) (built: Sep  6 2016 16:37:16) 
Copyright (c) 1997-2016 The PHP Group
Zend Engine v2.6.0, Copyright (c) 1998-2016 Zend Technologies

Instructor Tip:

For some of these processes, you may have the tool installed, but it is not in your PATH environment variable (the list of places your computer looks for programs to execute). You may need to track down the executable and then add the folder it's in to your PATH.

Composer

Install Composer on your local machine by following the Getting Started page.

Verify your installation by running the version command.


$ composer -V
Composer version 1.1.3 2016-06-26 15:42:08

Git

Install Git on your local machine by running the installation package from the Git Downloads page.

Verify your installation by running the version command.


$ git --version
git version 2.8.4 (Apple Git-73)

Congratulations

You now have all the tools in place to create your first Heroku application!
